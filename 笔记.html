AngularsJS 一. 1.概念 前端框架 以数据和逻辑为核心(驱动) 而Jquary是以DOM为核心(驱动) 2.设计模式:MVC模式 Model:模型层 主要用来处理数据,一般指操作数据库 View:视图层 主要用来展示数据,一般通过HTML展示 Controller:控制层 用来做连接模型层和视图层的桥梁 3.模块化:module 使用AnJS构建一个应用程序是以模块的方式组织的 编写分三步: 1)定义应用 ng-app="abc" 2)定义模块 var app = angulai.module("abc", []) 应用名称 数组(该模块所依赖的模块) 3)定义控制器 app.controller("ctrl", []) 控制器名称 数组(内含多个元素,前面的是字符串:angularJS自己定义好的;最后一个为function) 4.AngularsJS指令 已经定义好的指令(即内置指令) 都是以ng-开头 是AngularsJS的核心内容 ng-app : 定义应用,指定应用的根目录,至少有一个元素指定此属性 ng-model: 定义绑定数据的名称, ng-controller:定义控制器作用的范围 ng-bind: 数据绑定 ng-show/hide: 显示/隐藏数据 (ng-show用于显示/隐藏某个节点,值为1/true时显示,0/false时隐藏;ng-hide时相反) - ng-click: 单击 ng-if: 判断某个节点是否存在,存在-显示 - ng-repeat: 遍历,迭代 ng-src: ng-switch: ng-class: ng-disabled: 无法输入 - ng-readonly: - ng-checked: - ng-selected: - ng-include:用于引入另一个页面文件 注意点:使用ng-include时,页面是发送请求的,需要通过服务器访问 ng-include="'路径'" 请求路径需要放在*两对引号*之间 5.自定义指令 通过模块调用directive方法定义指令 定义好的指令有四种使用方式:(如果不加restrict,默认为E和A;添加之后,默认值无效) 1)元素 E(element) 2)属性 A(attribute) E和A是默认都可以使用的方法 3)类 C(class) 4)注释 M(mark) 需要添加restrict,设置添加C或M M还需要添加repalce:true,如果不添加指令会被真的注释掉 在使用注释方式时,需要有新的元素节点替换原来注释的地方 如果访问普通字符串:template url :需要使用templateUrl,同时要从服务器访问 二、常见事件 1、单击 ng-click 2、双击 ng-dblclick 3、鼠标移入 ng-mouseenter 4、鼠标移出 ng-mouseleave 5、失去焦点 ng-blur 6、文本框值的改变 ng-change (ng-change触发时，文本框内的内容是经过trim处理的) 三、数据绑定 1、单向数据绑定 2、双向数据绑定 3、数据的初始化 四、作用域 因为控制器最终跟HTML节点关联，而节点之间又存在包含关系， 导致控制器之间也有包含关系，最终导致$scope之间有不同的作用域。 跟作用域: $rootScope,跟app绑定，作用域的范围是整个app ng-init 跟在应用中，那么ng-init绑定在$rootScope上，范围就是整个app 子作用域: $scope,跟控制器绑定，作用域的范围就是控制器的范围 五、过滤器 1、AngularJs中使用过滤器进行格式化展示数据，在{{}}中使用 | (管道符号) 如果需要传递参数，使用 ：来分隔传递的参数 过滤器的使用还可以通过 9个内置过滤器: 1.currency : 将数字格式的数据格式化为货币格式 $5.0 ￥10.00 默认格式是$10.00 如果想要使用其他的货币符号，在currency后面添加参数(currency:"￥") 2.date : 格式化日期 3.filter : 从数组中选择一个子集,如果过滤的条件是一个对象，最后的 } 后需要加一个空格 4.json : 将对象转换为json格式 5.limitTo : 限制数组或字符串的长度 参数为正数时，从头开始截取;参数为负数时，从后面开始截取 6.number : 给数字添加千位分割符，如果没有参数，默认保留3位有效小数位 参数为正数，保留小数点后n位;参数为负数，保留小数点前n位 7.orderBy : 排序 无参数:默认排序,升序 字符串:将对象数组中按照对应的属性名进行排序，默认升序， 如果需要降序，添加第二个参数，true(false是默认的 升序) 在字符串前面加一个 + 表示升序， -
